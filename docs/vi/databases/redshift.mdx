# Kết nối Amazon Redshift

TablePro hỗ trợ Amazon Redshift, kho dữ liệu dạng cột của AWS dựa trên PostgreSQL 8.0.2. Redshift sử dụng giao thức PostgreSQL wire, nên các kết nối hoạt động qua cùng driver libpq với các truy vấn metadata riêng cho Redshift.

## Thiết lập nhanh

<Steps>
  <Step title="Mở Form kết nối">
    Click **New Connection** từ màn hình Welcome hoặc **File** > **New Connection**
  </Step>
  <Step title="Chọn Redshift">
    Chọn **Amazon Redshift** từ danh sách loại cơ sở dữ liệu
  </Step>
  <Step title="Nhập thông tin kết nối">
    Điền cluster endpoint, port, username, password và tên database
  </Step>
  <Step title="Kiểm tra và kết nối">
    Click **Test Connection**, sau đó **Create**
  </Step>
</Steps>

## Cài đặt kết nối

### Các trường bắt buộc

| Trường | Mô tả | Mặc định |
|-------|-------------|---------|
| **Name** | Tên định danh kết nối | - |
| **Host** | Cluster endpoint của Redshift | - |
| **Port** | Cổng server Redshift | `5439` |
| **Username** | Tên người dùng Redshift | - |
| **Password** | Mật khẩu người dùng | - |
| **Database** | Tên database để kết nối | `dev` |

<Warning>
Redshift yêu cầu chỉ định tên database để kết nối. Database mặc định được tạo với mỗi cluster là `dev`.
</Warning>

## Các cấu hình ví dụ

### Redshift Cluster

```
Name:     Production Warehouse
Host:     my-cluster.abc123xyz.us-east-1.redshift.amazonaws.com
Port:     5439
Username: awsuser
Password: (mật khẩu của bạn)
Database: dev
```

### Redshift Serverless

```
Name:     Serverless Warehouse
Host:     my-workgroup.123456789012.us-west-2.redshift-serverless.amazonaws.com
Port:     5439
Username: admin
Password: (mật khẩu của bạn)
Database: dev
```

<Tip>
Tìm cluster endpoint trong AWS Console tại Amazon Redshift > Clusters > cluster của bạn > Properties. Endpoint bao gồm port, nhưng hãy nhập host và port riêng biệt trong TablePro.
</Tip>

## Tính năng

### Hỗ trợ Schema

Redshift tổ chức các bảng thành schemas, giống như PostgreSQL. TablePro hiển thị:

1. Tất cả schemas có thể truy cập bởi người dùng của bạn
2. Các bảng trong mỗi schema
3. Thứ tự search path

### Chuyển đổi Schema

Chuyển đổi giữa các schema Redshift bằng trình chuyển đổi database (**Cmd+K**):

1. Nhấn **Cmd+K** để mở trình chuyển đổi
2. Chọn schema đích
3. Sidebar, truy vấn và toolbar sẽ cập nhật theo schema đã chọn

<Tip>
Schema mặc định là `public`. Các schema hệ thống như `pg_catalog` và `information_schema` được lọc khỏi danh sách.
</Tip>

### Duyệt bảng và xem dữ liệu

Duyệt bảng, views và late-binding views. Data grid hỗ trợ phân trang cho các bảng warehouse lớn. Metadata riêng của Redshift từ `svv_table_info` có sẵn trong panel thông tin bảng, bao gồm distribution style, sort keys và kích thước bảng.

### Thực thi truy vấn

Thực thi truy vấn với hỗ trợ cú pháp Redshift SQL. SQL editor cung cấp syntax highlighting cho các hàm và từ khóa riêng của Redshift.

```sql
-- Window functions
SELECT
    category,
    product,
    revenue,
    RANK() OVER (PARTITION BY category ORDER BY revenue DESC) as rank
FROM sales;

-- COPY từ S3
COPY sales FROM 's3://mybucket/data/sales/'
IAM_ROLE 'arn:aws:iam::123456789012:role/MyRedshiftRole'
FORMAT AS PARQUET;

-- UNLOAD ra S3
UNLOAD ('SELECT * FROM sales WHERE year = 2024')
TO 's3://mybucket/export/sales_2024_'
IAM_ROLE 'arn:aws:iam::123456789012:role/MyRedshiftRole'
FORMAT AS CSV;

-- Phân tích compression
ANALYZE COMPRESSION sales;
```

### Xuất dữ liệu

Xuất kết quả truy vấn hoặc dữ liệu bảng ở nhiều định dạng:

- CSV
- JSON
- SQL (câu lệnh INSERT)
- XLSX

### Nhập dữ liệu

Nhập dữ liệu từ file CSV, JSON, SQL và XLSX vào các bảng Redshift.

### Tạo DDL

Xem DDL (câu lệnh CREATE TABLE) cho bất kỳ bảng nào, bao gồm các mệnh đề riêng của Redshift như DISTKEY, SORTKEY, DISTSTYLE và các chỉ thị ENCODE.

### Điều hướng Foreign Key

Redshift hỗ trợ định nghĩa foreign key, mặc dù chúng chỉ mang tính thông tin và không được engine thực thi. TablePro hiển thị các mối quan hệ này và cho phép điều hướng giữa các bảng liên quan.

## SSL/TLS

Kết nối Redshift hỗ trợ mã hóa SSL/TLS, sử dụng cùng cấu hình như PostgreSQL:

| Chế độ SSL | Mô tả |
|----------|-------------|
| **Disabled** | Không mã hóa SSL |
| **Preferred** | Sử dụng SSL nếu có sẵn, quay lại không mã hóa nếu không |
| **Required** | Yêu cầu SSL, nhưng không xác minh chứng chỉ |
| **Verify CA** | Yêu cầu SSL và xác minh chứng chỉ server với CA |
| **Verify Identity** | Yêu cầu SSL, xác minh CA và xác minh hostname server |

<Tip>
Các endpoint AWS Redshift hỗ trợ SSL theo mặc định. Sử dụng chế độ **Required** hoặc **Verify CA** cho kết nối production.
</Tip>

## Hỗ trợ SSH Tunnel

Bạn có thể kết nối đến Redshift qua SSH tunnel nếu cluster của bạn nằm trong VPC private. Cấu hình SSH tunnel trong tab SSH của form kết nối, sau đó TablePro định tuyến kết nối Redshift qua tunnel.

## Hạn chế

Redshift là kho dữ liệu dạng cột, không phải RDBMS đa năng. Một số tính năng PostgreSQL không có hoặc hoạt động khác:

| Tính năng | Hành vi Redshift |
|---------|------------------|
| **Indexes** | Không có index B-tree truyền thống. Sử dụng DISTKEY và SORTKEY để phân phối và sắp xếp dữ liệu. Các cột này được hiển thị trong tab Structure. |
| **ALTER TABLE** | Hỗ trợ hạn chế. Không thể đổi tên cột. Thay đổi kiểu bị giới hạn ở mở rộng VARCHAR hoặc thay đổi độ chính xác số. |
| **Enum types** | Không hỗ trợ. Không có CREATE TYPE cho enum tùy chỉnh. |
| **Sequences** | Không hỗ trợ. Sử dụng cột IDENTITY thay thế. |
| **Foreign keys** | Chỉ mang tính thông tin. Được định nghĩa trong DDL nhưng không được engine thực thi. |
| **Triggers** | Không hỗ trợ. |
| **Stored procedures** | Hỗ trợ, nhưng với tập con PL/pgSQL bị giới hạn. |

### DISTKEY và SORTKEY

Thay vì index truyền thống, Redshift phân phối dữ liệu qua các node bằng distribution keys và sắp xếp dữ liệu trong mỗi node bằng sort keys. TablePro hiển thị các chỉ định DISTKEY và SORTKEY trong tab Structure cho mỗi bảng, giúp bạn thấy cách dữ liệu được tổ chức vật lý.

## Khắc phục sự cố

### Connection Refused

**Triệu chứng**: "Connection refused" hoặc timeout

**Nguyên nhân phổ biến**:

1. **Security group chưa cấu hình**: IP của bạn phải được cho phép trong VPC security group của Redshift cluster (inbound rule trên port 5439).
2. **Cluster không truy cập công khai**: Nếu `Publicly Accessible` được đặt là No trong thuộc tính cluster, bạn cần SSH tunnel hoặc VPN.
3. **Cluster đã tạm dừng**: Redshift cluster có thể được tạm dừng để tiết kiệm chi phí. Resume cluster trong AWS Console.

### Authentication Failed

**Triệu chứng**: "password authentication failed"

**Giải pháp**:

1. Xác minh master username và password trong AWS Console
2. Đối với xác thực IAM, đảm bảo thông tin đăng nhập còn hiệu lực
3. Kiểm tra người dùng đã được cấp quyền truy cập database được chỉ định

### Hiệu năng truy vấn

Redshift được tối ưu cho truy vấn phân tích trên tập dữ liệu lớn. Nếu truy vấn chậm:

1. Kiểm tra distribution style của bảng với `SELECT * FROM svv_table_info WHERE "table" = 'your_table'`
2. Chạy `ANALYZE` để cập nhật thống kê bảng
3. Sử dụng `EXPLAIN` để xem query plan
4. Đảm bảo các cột SORTKEY khớp với các predicate WHERE và JOIN thường dùng

## Khác biệt với PostgreSQL

Nếu bạn đến từ PostgreSQL:

| PostgreSQL | Redshift |
|------------|----------|
| Cổng mặc định 5432 | Cổng mặc định 5439 |
| Index B-tree, GIN, GiST | Chỉ DISTKEY + SORTKEY |
| `CREATE INDEX` | Không hỗ trợ |
| Thực thi constraint đầy đủ | Foreign key chỉ mang tính thông tin |
| `SERIAL` / `GENERATED AS IDENTITY` | `IDENTITY(seed, step)` |
| Hệ thống kiểu phong phú (JSONB, arrays, v.v.) | Kiểu hạn chế (không có arrays, JSON hạn chế qua kiểu SUPER) |
| `pg_stat_activity` | `stv_recents`, `svl_qlog` |
| Lưu trữ theo dòng | Lưu trữ theo cột |
